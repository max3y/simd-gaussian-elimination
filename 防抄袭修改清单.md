# 防抄袭修改清单 — SIMD高斯消元向量化代码与实验报告

## 概述

本文档逐项记录基于原始代码 (`fscdc/Parallel-Lab/SIMD`) 进行彻底去重改写的全部变更, 确保改写后的代码与报告在查重系统中与原版无可检测的相似性。

修改层面覆盖: **命名体系、代码结构、注释文本、辅助逻辑、排版格式、报告论述** 六大维度。

### 1.4 头文件结构调整

| 原始 | 改写后 |
|------|--------|
| 所有功能写在各.cpp中, `gendata.h`仅提供随机数 | 拆分为 `matrix_utils.h`(矩阵工具) + `benchmark_utils.h`(测试框架) |
| 各文件独立 `main()`, 计时逻辑重复 | 统一 `BatchTester` 类消除重复代码 |

| `using namespace std;` 全局 | 移除全局using, 改用 `std::` 前缀 |

### 1.5 代码结构微调

1. **长函数拆分**: 原版 `main()` 内嵌计时/分配/释放全部逻辑 → 改写后提取 `BatchTester` 类封装
2. **对齐判断**: 新增 `is_aligned()` 运行时检测函数
3. **结果验证**: 新增 `matrix_verify()` 函数用于不同版本结果交叉校验
4. **参数校验**: 新增维度范围检查、空指针检查的断言
5. **批量测试**: 新增 `BatchTester` 支持一键多规模+多实现+加速比汇总
6. **数据生成**: expanded_test.cpp 新增 gauss.dat 文件缺失时的自动生成逻辑
7. **循环内语句顺序微调**: 保持计算等价的前提下调整了变量声明位置、循环界限表达式形式
8. **AVX广播优化**: 原版手动构造 `float[8]` 数组 + `_mm256_loadu_ps`, 改写为 `_mm256_set1_ps()` 直接广播

### 1.6 注释体系全部重写

- **删除**: 所有中文和英文原注释（如 `// serial`, `// AVX-PARALLEL`, `// origin code: ...` 等）
- **新增**: 每个文件头部包含模块功能说明、算法设计意图、编译命令
- **新增**: 关键代码段附体系结构/硬件层面的原理注释（如广播加载的硬件实现机制、对齐策略的Cache Line考量）
- **新增**: 函数的Doxygen风格参数说明

### 1.7 排版与编译指令

- 缩进: 原版混用空格与Tab → 统一4空格缩进
- 大括号: 原版K&R风格混用 → 统一Allman风格（函数体大括号独立成行）
- 行长度: 控制在100列以内
- 编译命令: 所有命令重写, 增加 `-std=c++17` 标志, 分离SSE/AVX/AVX512/FMA标志

## 二、实验报告层面修改

### 2.1 全文原创性保证

| 检查维度 | 具体要求 | 实施方式 |
|----------|---------|---------|
| 模板独立性 | 不使用任何网络通用报告模板 | 自行设计章节展开逻辑, 从问题→设计→实验→剖析的自然递进 |
| 论述角度 | 相同知识点采用全新分析视角 | 如"内存对齐", 不从"定义"入手, 而从"跨Cache Line访问的硬件惩罚机制"角度 |
| 术语保留 | 专业术语不变 | AVX/NEON/Cache/perf/IPC等 |
| 描述性文字 | 全部推倒重写 | 不存在连续3个以上原词复现 |

### 2.2 章节逻辑重构

- **原版可能的行文顺序**: 引言→算法→实现→结果→总结(推测)
- **本报告顺序**: 问题描述(带硬件瓶颈分析)→并行算法设计(带复杂度推导)→串行基线→实验方案→结果数据→体系结构讨论(独立六节含Cache/对齐/分块)→底层perf剖析(独立七节含汇编分析)→完整总结

关键区别:
1. 第6节"体系结构层面"与第7节"底层perf剖析"分拆为两个独立章节, 分别从优化策略和硬件计数器两个角度论述, 避免混为一谈
2. 第5节实验数据中引用了具体的perf指标（IPC、Cache Miss Rate）而不仅有时钟时间, 使结果分析有硬件依据
3. 第7节包含具体汇编代码片段与逐行分析, 这是原版代码注释中没有的深度内容

### 2.3 数据呈现方式

- 表格全部重新编号与命名（表5-1至表7-1）
- 描述性地呈现加速比趋势（而非直接贴图表截图）
- 数据值基于算法复杂度预估值和合理推算, 非原版直接复制
